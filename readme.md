# アプリタイトル
タスク管理アプリケーション

# 機能詳細
- ログイン/ログアウト機能
- ユーザー登録(サインアップ)機能
- タスクを登録する機能
	- タイトル
	- ステータス(「未完了」「完了」から選択、登録時は自動的に「未完了」)
	- 優先度(「高」「中」「低」から選択)
	- 締切日時
	- 説明
- 登録したタスクを一覧表示する機能(「優先度が高い順」かつ「締切日時が早い順」、5件ごとのページネーション)
- ステータスまたは優先度によってタスクを絞り込み表示する機能
- タスクの詳細を表示する機能
	- ステータスが「完了」ならタイトルに打ち消し線が付き、灰色で表示
- タスクを編集する機能
- タスクを削除する機能

# 感想
取り掛かり始めた当初は期間内に完成させることができるか不安だったが、本棚アプリケーションをはじめとする授業資料や、インターネット上の記事を参考にしながら少しずつ進めるうちに理解が深まり、最終的にはとても楽しくアプリケーション開発を終えることができた。
いくつか「やりたかったがやり方がわからなかったもの」や「試してみたがうまくいかなかったもの」が残ってはいるものの、実装したかった機能や仕様の大半は実現することができた。また、前回よりもBootstrapを活用してデザイン面にもこだわって作成できたため、概ね満足である。
Djangoの機能に関して、今までやってきた中ではフォームを使用するのが特に苦手だと感じたため、いつか時間をとって学習したい。

# できればやりたかったこと
- 締切として登録されている日時と現在の日時を比較して、期限切れかつstatusがFalse(未完了)の場合、締切表示を赤文字にする。また、一覧表示では登録されている優先度にかかわらず一覧の一番上に表示する
- 一覧画面で、ステータスと優先度の2種類のフィルターを同時に適用して絞り込み表示する
- タスクの登録・編集時に、日時をカレンダーまたはプルダウンで選択して入力できるようにする
- タイトルによるタスク検索
- 説明によるタスク検索
